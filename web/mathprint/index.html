TI-84 Plus MathPrint renderer

Type an expression and it is typeset in 2-D using the calculator's real ROM font glyphs, following the page-0x39 MathPrint layout engine documented in the Equation display (MathPrint) wiki page. This is a reconstruction of the layout model, not the emulator; the glyph bitmaps are extracted from the ROM by tools/export-font.py.

step the emission order

This steps the calculator's real draw order: after a measure pass the OS emits whole glyphs and rules one at a time at pen coordinates (0x86D7/0x86D8), in emission order — large glyphs via _PutMap (07:4588), small ones via _VPutMap (01:6293), the fraction rule via eqdisp_draw_fraction_bar (39:6abf). Each step highlights the element drawn and the row below names the routine. Stretched structural marks (the ∫ stem, parentheses, vinculum) appear at the final step.

Syntax
  • a/b linear divide · a//b stacked fraction (small-font digits, like the n/d template)
  • a^b — exponent (small font; bottom sits 2 px below the base top)
  • sqrt(x) — radical (`2nd`+`x²`); nthroot(n,x) — nth root (`MATH→5`)
  • int(lo,hi,body,var) — definite integral (∫, `MATH→9`)
  • sum(var,lo,hi,body) — summation (Σ, `MATH→0`): hi above, var=lo below, body to the right
  • abs(x), parentheses, + - * ^, and implicit multiplication (2X)

Every construct here corresponds to a real TI-84 template (no invented ones). Geometry constants are ROM-confirmed: 5 px large-font cells, a 7 px fraction column unit (x = base + 7·col), a row_height + 2 px row step.

The ROM font glyphs extracted by tools/export-font.py, each drawn on its pixel grid. The large font is 5×7 cells on page 7 (drawn by _PutMap, 07:4588); the small/variable-width font is on page 3 (drawn by _VPutMap, 01:6293). Printable ASCII shows its character; other codes show their ROM token name.
